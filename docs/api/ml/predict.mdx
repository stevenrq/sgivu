## Descripción General

La API de Predicción ML proporciona pronósticos de demanda basados en Machine Learning para segmentos de vehículos. Predice la demanda mensual de ventas según las características del vehículo (marca, modelo, línea) utilizando modelos de series temporales entrenados con datos históricos de transacciones.

## URL Base

```
https://your-domain.com/v1/ml
```

---

## Predecir Demanda

```http
POST /v1/ml/predict
```

Genera un pronóstico de demanda para un segmento de vehículo específico.

**Autenticación:** Requerida (JWT o clave de servicio interno)

**Autorización:** Permisos configurables (por defecto: requiere token válido)

### Cuerpo de la Solicitud

<ParamField body="vehicle_type" type="string" required>
  Tipo de vehículo: `CAR` o `MOTORCYCLE`
</ParamField>

<ParamField body="brand" type="string" required>
  Marca/fabricante del vehículo (ej., "Toyota", "Yamaha")
</ParamField>

<ParamField body="model" type="string" required>
  Año del modelo (ej., "2022", "2023")
</ParamField>

<ParamField body="line" type="string" required>
  Línea/variante del vehículo (ej., "Corolla", "MT-09")
</ParamField>

<ParamField body="horizon_months" type="integer" default="6">
  Horizonte de pronóstico en meses (1-24)
</ParamField>

<ParamField body="confidence" type="number" default="0.95">
  Nivel de confianza para los intervalos de predicción (0.5-0.99)
</ParamField>

```json
{
  "vehicle_type": "CAR",
  "brand": "Toyota",
  "model": "2022",
  "line": "Corolla",
  "horizon_months": 6,
  "confidence": 0.95
}
```

### Respuesta

<ResponseField name="predictions" type="array">
  Arreglo de predicciones mensuales
</ResponseField>

<ResponseField name="predictions[].month" type="string">
  Mes en formato YYYY-MM
</ResponseField>

<ResponseField name="predictions[].demand" type="number">
  Demanda predicha (estimación puntual)
</ResponseField>

<ResponseField name="predictions[].lower_ci" type="number">
  Límite inferior del intervalo de confianza
</ResponseField>

<ResponseField name="predictions[].upper_ci" type="number">
  Límite superior del intervalo de confianza
</ResponseField>

<ResponseField name="model_version" type="string">
  Identificador de versión del modelo utilizado
</ResponseField>

<ResponseField name="metrics" type="object" nullable>
  Métricas de rendimiento del modelo (MAE, RMSE, etc.)

### Códigos de Estado

- `200 OK` - Predicción generada exitosamente
- `400 Bad Request` - Parámetros de entrada inválidos
- `401 Unauthorized` - Autenticación faltante o inválida
- `403 Forbidden` - Permisos insuficientes
- `404 Not Found` - Modelo no disponible o segmento no encontrado
- `500 Internal Server Error` - Falló la predicción del modelo

### Ejemplo

```bash
curl -X POST https://your-domain.com/v1/ml/predict \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "vehicle_type": "CAR",
    "brand": "Toyota",
    "model": "2022",
    "line": "Corolla",
    "horizon_months": 6,
    "confidence": 0.95
```

## Interpretación de Predicciones

### Estimación Puntual

El campo `demand` representa la mejor estimación del modelo sobre las ventas mensuales.

### Intervalos de Confianza

Los campos `lower_ci` y `upper_ci` proporcionan intervalos de predicción:

- Con `confidence=0.95` (por defecto), existe un 95% de probabilidad de que la demanda real se encuentre dentro de `[lower_ci, upper_ci]`
- Intervalos más amplios indican mayor incertidumbre
- Los intervalos típicamente se amplían para horizontes de pronóstico más largos

### Ejemplo de Interpretación

```json
{
  "month": "2026-04",
  "demand": 8.5,
  "lower_ci": 5.2,
  "upper_ci": 11.8
}
```

**Interpretación:** El modelo predice que se venderán 8-9 unidades en abril de 2026, con un 95% de confianza de que las ventas reales estarán entre 5 y 12 unidades.

---

## Reglas de Validación

### Tipo de Vehículo

Debe ser exactamente `CAR` o `MOTORCYCLE` (sensible a mayúsculas/minúsculas).

### Campo Line

El campo `line` es **obligatorio** y no debe estar vacío después de eliminar espacios en blanco.

### Meses de Horizonte

- **Mínimo:** 1 mes
- **Máximo:** 24 meses
- **Por defecto:** 6 meses

### Nivel de Confianza

- **Mínimo:** 0.5 (50%)
- **Máximo:** 0.99 (99%)
- **Por defecto:** 0.95 (95%)

## Métricas del Modelo

Cuando están disponibles, la respuesta incluye métricas de rendimiento del modelo:

<ResponseField name="mae" type="number">
  Mean Absolute Error - Magnitud promedio del error de predicción
</ResponseField>

<ResponseField name="rmse" type="number">
  Root Mean Square Error - Desviación estándar de los errores de predicción
</ResponseField>

<ResponseField name="mape" type="number">
  Mean Absolute Percentage Error - Error promedio como porcentaje
</ResponseField>

**Valores más bajos indican mayor precisión del modelo.**

## Respuestas de Error

### 400 Bad Request - Tipo de Vehículo Inválido

```json
{
  "detail": [
    {
      "loc": ["body", "vehicle_type"],
      "msg": "vehicle_type must be CAR or MOTORCYCLE",
      "type": "value_error"
    }
  ]
}
```

### 400 Bad Request - Horizonte Inválido

```json
{
  "detail": [
    {
      "loc": ["body", "horizon_months"],
      "msg": "ensure this value is less than or equal to 24",
      "type": "value_error.number.not_le"
    }
  ]
}
```

### 404 Not Found - Modelo No Disponible

```json
{
  "detail": "No trained model available. Please train a model first."
}
```

### 404 Not Found - Segmento No Encontrado

```json
{
  "detail": "No historical data found for the specified vehicle segment"
}
```

### 500 Internal Server Error

```json
{
  "detail": "Model prediction failed: insufficient training data"
}
```

---

## Notas de Rendimiento

- Las predicciones se generan típicamente en < 500ms
- Las consultas de datos históricos pueden tardar más para segmentos con historial extenso
- Considere cachear predicciones para segmentos consultados frecuentemente
- Las predicciones son deterministas para la misma versión del modelo y los mismos datos de entrada
