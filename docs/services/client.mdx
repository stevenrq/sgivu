## Descripción General

`sgivu-client` gestiona la información de clientes (personas y empresas) dentro del ecosistema SGIVU. Proporciona operaciones CRUD, búsqueda avanzada y contadores, exponiendo una API REST protegida por roles y permisos emitidos por el Servidor de Autorización (`sgivu-auth`).

## Tecnologías

- **Java**: 21
- **Spring Boot**: 4.0.1
- **Spring Security**: Resource Server (validación JWT)
- **Spring Data JPA**: Persistencia con PostgreSQL
- **Flyway**: Migraciones de base de datos en `src/main/resources/db/migration`
- **Spring Boot Actuator**: Monitoreo de salud
- **Micrometer Tracing**: Integración con Zipkin
- **SpringDoc OpenAPI**: Documentación de la API
- **MapStruct**: Mapeo de DTOs
- **Lombok**: Generación de código

## Configuración de Puerto

- **Puerto por defecto**: 8082 (configurable vía `PORT` o config server)

## Prerrequisitos

- JDK 21
- Maven 3.9+
- Base de datos PostgreSQL
- Servicios `sgivu-config` y `sgivu-discovery` en ejecución

## Ejecución del Servicio

```bash
cd infra/compose/sgivu-docker-compose
docker compose -f docker-compose.dev.yml up -d
```

## Tipos de Cliente

### Persona

**Clientes individuales** con información personal:
- Nombre, apellido
- Número de documento (cédula, pasaporte, etc.)
- Fecha de nacimiento
- Correo electrónico, teléfono
- Direcciones

### Empresa

**Clientes empresariales** con información corporativa:
- Razón social
- Identificación tributaria (NIT/RUC)
- Sector industrial
- Correo electrónico, teléfono
- Direcciones

## Seguridad

### Autenticación

**Validación de Token JWT**:
- `sgivu-client` actúa como **Resource Server**
- Valida tokens emitidos por `sgivu-auth`
- La firma del token se verifica mediante el endpoint JWKS

### Autorización

**Permisos Requeridos**:
- `person:create` - Crear clientes persona
- `person:read` - Consultar clientes persona
- `person:update` - Actualizar clientes persona
- `person:delete` - Eliminar clientes persona
- `company:create` - Crear clientes empresa
- `company:read` - Consultar clientes empresa
- `company:update` - Actualizar clientes empresa
- `company:delete` - Eliminar clientes empresa

**Implementación**: Anotaciones `@PreAuthorize` en `PersonController` y `CompanyController`

<Info>
El Gateway o los clientes deben enviar un token Bearer válido con las autoridades requeridas. El token debe ser emitido por `sgivu-auth`.
</Info>

## Esquema de Base de Datos

### Migraciones Flyway

**Ubicación**: `src/main/resources/db/migration/V1__initial_schema.sql`

**Tablas Creadas**:

<Tabs>
  <Tab title="clients">
    Tabla base de clientes con discriminador:
    - `id`: Clave primaria
    - `client_type`: ENUM (PERSON, COMPANY)
    - `email`, `phone`: Información de contacto
    - `created_at`, `updated_at`: Marcas de tiempo de auditoría
  </Tab>
  
  <Tab title="persons">
    Información específica de personas:
    - `client_id`: Clave foránea a clients
    - `first_name`, `last_name`
    - `document_number`: Identificador único
    - `date_of_birth`
  </Tab>
  
  <Tab title="companies">
    Información específica de empresas:
    - `client_id`: Clave foránea a clients
    - `legal_name`
    - `tax_id`: Identificador único
    - `industry_sector`
  </Tab>
  
  <Tab title="addresses">
    Direcciones físicas:
    - `id`: Clave primaria
    - `client_id`: Clave foránea a clients
    - `street`, `city`, `state`, `country`
    - `postal_code`
    - `address_type`: ENUM (HOME, WORK, BILLING, SHIPPING)
  </Tab>
</Tabs>

**Índices**:
- `persons.document_number` (único)
- `companies.tax_id` (único)
- `clients.email`
- `addresses.client_id`

### Configuración

Conexión a la base de datos configurada en `sgivu-config-repo/sgivu-client-*.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://${DB_HOST}:${DB_PORT}/${DB_NAME}
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
  flyway:
    enabled: true
    locations: classpath:db/migration
```

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Errores de autorización 401/403">
    **Síntomas**: Las peticiones a la API retornan no autorizado o prohibido
    
    **Soluciones**:
    - Verificar que el token Bearer esté presente en el header Authorization
    - Comprobar que el token contiene las autoridades requeridas
    - Asegurar que el emisor del token sea `sgivu-auth`
    - Validar que el token no haya expirado
    - Revisar las asignaciones de roles del usuario en el Servicio de Usuarios
  </Accordion>
  
  <Accordion title="Errores de conexión a la base de datos">
    **Síntomas**: El servicio falla al iniciar o las operaciones de base de datos fallan
    
    **Soluciones**:
    - Verificar las variables de entorno `DEV_CLIENT_DB_*` o `PROD_CLIENT_DB_*`
    - Comprobar que PostgreSQL esté corriendo y accesible
    - Asegurar que la base de datos y el usuario existan
    - Revisar el estado de las migraciones Flyway
    - Verificar las credenciales de la base de datos
  </Accordion>
  
  <Accordion title="Fallos en migraciones Flyway">
    **Síntomas**: El servicio falla al iniciar con un error de Flyway
    
    **Soluciones**:
    - Verificar la sintaxis del script de migración
    - Comprobar que el usuario de la base de datos tenga permisos CREATE/ALTER
    - Revisar la tabla `flyway_schema_history`
    - Limpiar migraciones fallidas si está en desarrollo
    - Asegurar que las migraciones sean secuenciales (V1, V2, etc.)
  </Accordion>
  
  <Accordion title="Error de documento/identificación tributaria duplicada">
    **Síntomas**: La creación falla con violación de restricción única
    
    **Soluciones**:
    - Verificar si el cliente ya existe antes de la creación
    - Manejar las excepciones de clave duplicada adecuadamente
    - Retornar un mensaje de error claro al usuario
    - Considerar la eliminación lógica para clientes desactivados
    - Permitir búsqueda por documento antes de la creación
  </Accordion>
  
  <Accordion title="El servicio no aparece en Eureka">
    **Síntomas**: El servicio de clientes no es visible en el dashboard de Eureka
    
    **Soluciones**:
    - Verificar la configuración de `eureka.client.service-url.defaultZone`
    - Comprobar la conectividad de red con `sgivu-discovery`
    - Asegurar que `spring.application.name` esté configurado correctamente
    - Revisar los logs de registro en Eureka
    - Esperar 30-60 segundos para que se complete el registro
  </Accordion>
</AccordionGroup>

## Variables de Entorno

| Variable | Descripción | Ejemplo |
|----------|-------------|----------|
| `PORT` | Puerto del servidor | `8082` |
| `DB_HOST` | Host de PostgreSQL | `sgivu-postgres-client` |
| `DB_PORT` | Puerto de PostgreSQL | `5432` |
| `DB_NAME` | Nombre de la base de datos | `sgivu_client` |
| `DB_USERNAME` | Usuario de la base de datos | `sgivu_user` |
| `DB_PASSWORD` | Contraseña de la base de datos | `secure_password` |
| `DEV_CLIENT_DB_*` | Variables de base de datos de desarrollo | Prefijadas con `DEV_CLIENT_DB_` |
| `PROD_CLIENT_DB_*` | Variables de base de datos de producción | Prefijadas con `PROD_CLIENT_DB_` |
| `SPRING_SECURITY_OAUTH2_RESOURCESERVER_JWT_ISSUER_URI` | Emisor del servidor de autorización | `http://sgivu-auth:9000` |
