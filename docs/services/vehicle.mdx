## Descripción General

`sgivu-vehicle` gestiona el catálogo de vehículos (automóviles y motocicletas) y sus imágenes asociadas. El servicio expone **dos APIs independientes por tipo de vehículo** — `/v1/cars` y `/v1/motorcycles` — además de una API para imágenes bajo `/v1/vehicles/{vehicleId}/images`. Soporta operaciones CRUD, búsqueda avanzada multi-criterio, cambio de estado y carga/gestión de imágenes mediante AWS S3 con URLs prefirmadas.

## Integración con AWS S3

### Servicio S3

**S3Service** proporciona generación de URLs prefirmadas para:
- **Carga**: Subidas directas del cliente a S3
- **Descarga**: Acceso temporal a imágenes

**Beneficios**:
- Reduce la carga del servidor (subida directa)
- Acceso temporal seguro
- Sin consumo de ancho de banda del proxy

### Flujo de URLs Prefirmadas

<Steps>
  <Step title="Solicitar URL de Carga">
    El cliente solicita una URL prefirmada al Backend:
    ```bash
    POST /v1/vehicles/{vehicleId}/images/presigned-upload
    {
      "contentType": "image/jpeg"
    }
    ```
  </Step>

  <Step title="Subir a S3">
    El cliente sube directamente a S3 usando la URL prefirmada devuelta:
    ```bash
    PUT https://bucket.s3.amazonaws.com/images/vehicle-123-<timestamp>.jpg
    Content-Type: image/jpeg

    [binary image data]
    ```
  </Step>

  <Step title="Confirmar Carga">
    El cliente confirma la carga al Backend con los metadatos:
    ```bash
    POST /v1/vehicles/{vehicleId}/images/confirm-upload
    {
      "fileName": "front-view.jpg",
      "contentType": "image/jpeg",
      "size": 245760,
      "key": "images/vehicle-123-<timestamp>.jpg",
      "primary": true
    }
    ```
    El Backend valida la clave en S3 y registra los metadatos de la imagen en la base de datos.
  </Step>

  <Step title="Listar Imágenes">
    El cliente obtiene todas las imágenes de un vehículo:
    ```bash
    GET /v1/vehicles/{vehicleId}/images
    ```
    La respuesta incluye la `url` (prefirmada o pública según configuración) de cada imagen.
  </Step>
</Steps>

### Tipos de Imagen Permitidos

Solo se aceptan estos tipos MIME:
- `image/jpeg`
- `image/png`
- `image/webp`

<Warning>
Las solicitudes de carga con otros tipos de contenido serán rechazadas. Valide el tipo de contenido en el cliente antes de solicitar la URL prefirmada.
</Warning>

### Gestión de Imagen Principal

- **Primera Imagen**: Se marca automáticamente como `is_primary=true`
- **Eliminar Principal**: La siguiente imagen más antigua se promueve automáticamente a principal
- **Establecer Principal**: Use el Endpoint dedicado para cambiar la imagen principal

## Configuración CORS

### CORS del Bucket

**S3BucketCorsConfig** asegura que el bucket S3 permita cargas desde los orígenes configurados:

```java
@Configuration
public class S3BucketCorsConfig {
    @PostConstruct
    public void configureBucketCors() {
        // Reads from aws.s3.allowed-origins
        // Applies CORS rules to bucket
    }
}
```

**Configuración**:
```yaml
aws:
  s3:
    bucket-name: sgivu-vehicles
    allowed-origins:
      - http://localhost:3000
      - https://app.sgivu.com
```

<Info>
Actualice `allowed-origins` para que coincida con los dominios de su Frontend en producción.
</Info>

## Seguridad

### Autenticación

**Validación de Tokens JWT**:
- Tokens emitidos por `sgivu-auth`
- `JwtAuthenticationConverter` mapea claims a authorities
- El Resource Server valida la firma mediante JWKS

### Autorización

**Permisos Requeridos** (aplicados vía `@PreAuthorize` en controllers):

- `car:create`, `car:read`, `car:update`, `car:delete` — API de autos (`/v1/cars`)
- `motorcycle:create`, `motorcycle:read`, `motorcycle:update`, `motorcycle:delete` — API de motocicletas (`/v1/motorcycles`)
- `vehicle:create`, `vehicle:read`, `vehicle:update`, `vehicle:delete` — API de imágenes (`/v1/vehicles/{vehicleId}/images`)

<Info>
No existe un permiso único `vehicle:*` para CRUD: autos y motocicletas tienen sus propios dominios de permisos. Los permisos `vehicle:*` se usan exclusivamente en la API de imágenes.
</Info>

### Comunicación Interna entre Servicios

**InternalServiceAuthenticationFilter**:
- Permite llamadas entre servicios mediante `X-Internal-Service-Key`
- Omite la validación JWT para Endpoints internos
- Debe mantenerse en secreto y rotarse periódicamente

### Recomendaciones de Seguridad AWS

<Warning>
**Seguridad en Producción**:
- Mover credenciales AWS a un gestor de secretos (AWS Secrets Manager, HashiCorp Vault)
- Usar roles IAM en lugar de access keys al ejecutar en EC2/ECS
- Aplicar políticas de bucket con privilegios mínimos
- Limitar el tiempo de expiración de URLs prefirmadas (actualmente 15 minutos)
- Habilitar versionado y logging del bucket S3
- Usar VPC Endpoints para acceso a S3
</Warning>

## Esquema de Base de Datos

### Migraciones Flyway

**Ubicación**: `src/main/resources/db/migration`

**Tablas Creadas**:
- `vehicles`: Catálogo principal de vehículos
  - Campos comunes: VIN, marca, modelo, año, precio, estado
  - Discriminador de tipo para automóviles vs motocicletas
- `vehicle_images`: Metadatos de imágenes
  - Clave S3, nombre de archivo, tipo de contenido
  - Flag `is_primary`
  - Clave foránea a vehicles

**Índices**:
- `vehicles.vin` (unique)
- `vehicles.status`
- `vehicles.vehicle_type`
- `vehicle_images.vehicle_id`
- `vehicle_images.is_primary`

## Endpoints de la API

### Autos (`/v1/cars`)

<ResponseField name="POST /v1/cars" type="protected">
  Alta de auto.

  **Permiso Requerido**: `car:create`
</ResponseField>

<ResponseField name="GET /v1/cars/{id}" type="protected">
  Obtiene detalle de un auto.

  **Permiso Requerido**: `car:read`
</ResponseField>

<ResponseField name="GET /v1/cars/page/{page}" type="protected">
  Listado paginado de autos.

  **Permiso Requerido**: `car:read`
</ResponseField>

<ResponseField name="PUT /v1/cars/{id}" type="protected">
  Actualiza un auto por id.

  **Permiso Requerido**: `car:update`
</ResponseField>

<ResponseField name="DELETE /v1/cars/{id}" type="protected">
  Elimina un auto por id.

  **Permiso Requerido**: `car:delete`
</ResponseField>

<ResponseField name="PATCH /v1/cars/{id}/status" type="protected">
  Cambia el estado del auto (body: enum `VehicleStatus`).

  **Permiso Requerido**: `car:update`
</ResponseField>

<ResponseField name="GET /v1/cars/count" type="protected">
  Conteo de autos agrupado por estado.

  **Permiso Requerido**: `car:read`
</ResponseField>

<ResponseField name="GET /v1/cars/search" type="protected">
  Busca autos con múltiples filtros opcionales: `plate`, `brand`, `line`, `model`, `fuelType`, `bodyType`, `transmission`, `city`, `status`, `minYear`, `maxYear`, `minCapacity`, `maxCapacity`, `minMileage`, `maxMileage`, `minSalePrice`, `maxSalePrice`.

  **Permiso Requerido**: `car:read`
</ResponseField>

<ResponseField name="GET /v1/cars/search/page/{page}" type="protected">
  Versión paginada de la búsqueda anterior (acepta además `size`).

  **Permiso Requerido**: `car:read`
</ResponseField>

### Motocicletas (`/v1/motorcycles`)

El recurso `/v1/motorcycles` ofrece el mismo conjunto de operaciones que `/v1/cars` (CRUD + paginado + `/count` + `/search` + `/search/page/{page}`) pero con permisos `motorcycle:*` y filtros específicos: `plate`, `brand`, `line`, `model`, `motorcycleType`, `transmission`, `city`, `status`, `minYear`/`maxYear`, `minCapacity`/`maxCapacity` (cilindraje), `minMileage`/`maxMileage`, `minSalePrice`/`maxSalePrice`.

### Imágenes (`/v1/vehicles/{vehicleId}/images`)

<ResponseField name="POST /v1/vehicles/{vehicleId}/images/presigned-upload" type="protected">
  Genera URL prefirmada para subir una imagen a S3.

  **Permiso Requerido**: `vehicle:create`

  **Body**:
  ```json
  {
    "contentType": "image/jpeg"
  }
  ```
</ResponseField>

<ResponseField name="POST /v1/vehicles/{vehicleId}/images/confirm-upload" type="protected">
  Confirma la subida y registra los metadatos en PostgreSQL.

  **Permiso Requerido**: `vehicle:create`

  **Body**:
  ```json
  {
    "fileName": "front-view.jpg",
    "contentType": "image/jpeg",
    "size": 245760,
    "key": "images/vehicle-123-<timestamp>.jpg",
    "primary": true
  }
  ```
</ResponseField>

<ResponseField name="GET /v1/vehicles/{vehicleId}/images" type="protected">
  Lista las imágenes del vehículo con sus metadatos y URLs.

  **Permiso Requerido**: `vehicle:read`
</ResponseField>

<ResponseField name="DELETE /v1/vehicles/{vehicleId}/images/{imageId}" type="protected">
  Elimina una imagen del vehículo (tanto de PostgreSQL como de S3).

  **Permiso Requerido**: `vehicle:delete`

  **Comportamiento**: Si se elimina la imagen marcada como principal, la siguiente imagen más antigua se promueve automáticamente a principal.
</ResponseField>

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Errores de autorización 401/403">
    **Síntomas**: Las solicitudes a la API retornan no autorizado o prohibido

    **Soluciones**:
    - Verificar que el token Bearer contenga las authorities requeridas
    - Comprobar que el issuer del token coincida con el servidor de autenticación
    - Asegurar que los roles del usuario incluyan permisos de vehículos
    - Validar que el token no haya expirado
  </Accordion>

  <Accordion title="Fallos en la carga a S3">
    **Síntomas**: La carga con URL prefirmada retorna 403 o falla

    **Soluciones**:
    - Verificar credenciales y permisos de AWS
    - Comprobar que la política del bucket S3 permita PutObject
    - Confirmar que el nombre del bucket sea correcto
    - Validar que el tipo de contenido esté permitido
    - Revisar la configuración CORS del bucket
    - Verificar que la URL prefirmada no haya expirado
  </Accordion>

  <Accordion title="La imagen no se sube correctamente">
    **Síntomas**: La carga a S3 tiene éxito pero la imagen no aparece

    **Soluciones**:
    - Asegurar que se llame al Endpoint confirm-upload
    - Verificar que la clave S3 coincida entre la carga y la confirmación
    - Comprobar el registro de imagen en la base de datos
    - Revisar los logs del Backend en busca de errores de validación
    - Confirmar que el ID del vehículo exista
  </Accordion>

  <Accordion title="Errores CORS durante la carga">
    **Síntomas**: El navegador bloquea la carga a S3 con error CORS

    **Soluciones**:
    - Verificar que `aws.s3.allowed-origins` incluya el dominio del Frontend
    - Comprobar que la configuración CORS del bucket S3 se haya aplicado
    - Asegurar que `S3BucketCorsConfig` se ejecutó al iniciar
    - Agregar reglas CORS explícitas en la consola de AWS si es necesario
    - Verificar que el header Content-Type coincida con los tipos permitidos
  </Accordion>

  <Accordion title="La imagen principal no se establece correctamente">
    **Síntomas**: Se muestra una imagen incorrecta como principal o no hay imagen principal

    **Soluciones**:
    - Verificar la consistencia del flag `is_primary` en la base de datos
    - Comprobar que solo una imagen por vehículo tenga `is_primary=true`
    - Revisar la lógica de eliminación para la promoción de imagen principal
    - Reenviar el `confirm-upload` con `primary: true` para reasignar la principal (no existe endpoint `set-primary` dedicado)
  </Accordion>

  <Accordion title="Error de placa duplicada">
    **Síntomas**: La creación del vehículo falla con violación de restricción única

    **Soluciones**:
    - Verificar la unicidad de la placa (`plate`) antes de la creación
    - Manejar la excepción de clave duplicada correctamente
    - Considerar eliminación lógica para vehículos retirados
  </Accordion>
</AccordionGroup>
