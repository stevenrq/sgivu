El servicio Gateway (`sgivu-gateway`) es el punto de entrada para todas las solicitudes de los clientes, encargándose de la autenticación OAuth2, la gestión de sesiones y el enrutamiento hacia los servicios backend.

## Características Principales

- API Gateway reactivo construido con Spring Cloud Gateway
- Cliente OAuth2 con flujo de código de autorización
- Gestión de sesiones distribuidas basada en Redis
- Token Relay hacia servicios downstream
- Revocación de sesiones y manejo de cierre de sesión
- Integración con descubrimiento de servicios

## Configuración Base

### Configuración del Servidor

```yaml
server:
  port: ${PORT:8080}
  forward-headers-strategy: framework
```

<Info>
El gateway utiliza WebFlux reactivo para el manejo no bloqueante de solicitudes.
</Info>

## Gestión de Sesiones

El gateway utiliza Redis para el almacenamiento distribuido de sesiones:

```yaml
spring:
  session:
    store-type: redis
    timeout: 1h
    redis:
      namespace: spring:session:sgivu-gateway
  data:
    redis:
      host: ${REDIS_HOST:sgivu-redis}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD}
```

<Accordion title="Beneficios de Sesiones en Redis">
- **Escalado horizontal**: Múltiples instancias del gateway pueden compartir sesiones
- **Acceso rápido**: Rendimiento en memoria para consultas de sesión
- **Expiración automática**: El TTL de Redis gestiona la limpieza de sesiones
- **Persistencia**: Las sesiones sobreviven a reinicios del gateway
</Accordion>

<Warning>
Asegúrese de que Redis tenga configurada la persistencia adecuada (RDB o AOF) para evitar la pérdida de sesiones ante un reinicio de Redis.
</Warning>

## Configuración del Cliente OAuth2

### Registro del Cliente

```yaml
spring:
  security:
    oauth2:
      client:
        registration:
          sgivu-gateway:
            provider: sgivu-auth
            client-id: sgivu-gateway
            client-secret: ${SGIVU_GATEWAY_SECRET}
            client-authentication-method: client_secret_basic
            authorization-grant-type: authorization_code
            redirect-uri: "{baseUrl}/login/oauth2/code/{registrationId}"
            scope:
              - openid
              - profile
              - email
              - phone
              - address
              - offline_access
              - api
              - read
              - write
```

<Note>
El scope `offline_access` habilita la emisión de refresh tokens para la renovación transparente de tokens.
</Note>

### Configuración del Proveedor

```yaml
spring:
  security:
    oauth2:
      client:
        provider:
          sgivu-auth:
            issuer-uri: ${SGIVU_AUTH_URL:http://sgivu-auth:9000}
```

La URI del issuer permite el descubrimiento automático de los endpoints de autorización a través del documento de descubrimiento de OpenID Connect.

## Token Relay

El gateway retransmite automáticamente los tokens de acceso OAuth2 a los servicios downstream. Esto permite:

- Inicio de sesión único (SSO) entre servicios
- Identidad de usuario consistente
- Renovación centralizada de tokens
- Propagación automática de tokens

```yaml
# Token relay is configured automatically by Spring Cloud Gateway
# when OAuth2 client is present
```

spring:

</Info>

## Configuración de Rutas

El gateway enruta las solicitudes a los servicios backend según patrones de ruta:

- `/api/auth/**` → Servicio Auth
- `/api/users/**` → Servicio User
- `/api/clients/**` → Servicio Client
- `/api/vehicles/**` → Servicio Vehicle
- `/api/purchase-sales/**` → Servicio Purchase-Sale

<Note>
La configuración de rutas se define típicamente en el código de la aplicación del gateway, no en la configuración externa.
</Note>

## Características de Seguridad

### Revocación de Sesiones

El gateway soporta cierre de sesión y revocación de tokens:
- Eliminación local de la sesión en Redis
- Revocación de tokens en el servidor de autorización
- Redirección a la página de inicio de sesión

### Protección CSRF

La protección CSRF está habilitada para todas las operaciones que modifican estado.

### Renovación de Tokens

La renovación automática de tokens mediante refresh tokens previene la interrupción de sesiones.
