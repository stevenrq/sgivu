El Servicio de Clientes (`sgivu-client`) gestiona las cuentas, perfiles y operaciones de negocio relacionadas con los clientes en la plataforma SGIVU.

## Características Principales

- Gestión de cuentas de clientes
- Perfiles e información de contacto de clientes
- Integración con el Servicio de Compra-Venta
- Migración de base de datos con Flyway
- API RESTful con documentación OpenAPI
- Autenticación interna entre servicios

## Configuración Base

### Configuración del Servidor

```yaml
server:
  port: ${PORT:8082}
```

### Configuración de JPA

```yaml
spring:
  jpa:
    open-in-view: false
```

<Info>
Deshabilitar open-in-view previene excepciones de carga diferida al forzar los límites de las transacciones.
</Info>

### Migración con Flyway

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: ${FLYWAY_BASELINE_ON_MIGRATE:false}
    baseline-version: 0
    validate-on-migrate: true
```

<Accordion title="Configuración de Flyway">
- **enabled: true**: Ejecuta las migraciones automáticamente al iniciar
- **locations**: Ubicación donde se almacenan los scripts de migración
- **baseline-on-migrate**: Gestiona bases de datos existentes sin historial de versiones
- **baseline-version: 0**: Versión inicial para el baseline
- **validate-on-migrate: true**: Garantiza la integridad de las migraciones
</Accordion>

## Configuración de Base de Datos

<Tabs>
  <Tab title="Desarrollo">
    ```yaml
    spring:
      datasource:
        url: jdbc:postgresql://${DEV_CLIENT_DB_HOST:host.docker.internal}:${DEV_CLIENT_DB_PORT:5432}/${DEV_CLIENT_DB_NAME}
        username: ${DEV_CLIENT_DB_USERNAME}
        password: ${DEV_CLIENT_DB_PASSWORD}
        driver-class-name: org.postgresql.Driver
      jpa:
        hibernate:
          ddl-auto: validate
        show-sql: true
        properties:
          hibernate:
            format_sql: true
      flyway:
        locations: classpath:db/migration
        baseline-on-migrate: true
        clean-disabled: true
    ```
    
    **Características de Desarrollo:**
    - Registro de SQL con formato para depuración
    - Baseline on migrate para bases de datos existentes
    - Clean deshabilitado (cambiado de false) por seguridad
  </Tab>
  <Tab title="Producción">
    ```yaml
    spring:
      datasource:
        url: jdbc:postgresql://${PROD_CLIENT_DB_HOST}:${PROD_CLIENT_DB_PORT}/${PROD_CLIENT_DB_NAME}
        username: ${PROD_CLIENT_DB_USERNAME}
        password: ${PROD_CLIENT_DB_PASSWORD}
        driver-class-name: org.postgresql.Driver
      jpa:
        hibernate:
          ddl-auto: validate
      flyway:
        clean-disabled: true
        baseline-on-migrate: ${FLYWAY_BASELINE_ON_MIGRATE:false}
    ```
    
    **Seguridad en Producción:**
    - Sin registro de SQL para mejorar el rendimiento
    - Clean deshabilitado para prevenir pérdida accidental de datos
    - Migración baseline controlada
  </Tab>
</Tabs>

<Warning>
Tenga en cuenta que incluso en desarrollo, `clean-disabled: true` previene la eliminación accidental de la base de datos. Tenga precaución al modificar esta configuración.
</Warning>

## Integración de Servicios

### Registro en Eureka

```yaml
eureka:
  instance:
    instance-id: ${spring.cloud.client.hostname}:${spring.application.name}:${random.value}
  client:
    service-url:
      defaultZone: ${EUREKA_URL:http://sgivu-discovery:8761/eureka}
```

<Note>
El ID de instancia aleatorio permite ejecutar múltiples instancias para escalado horizontal.
</Note>

### Mapa de Descubrimiento de Servicios

```yaml
services:
  map:
    sgivu-auth:
      name: sgivu-auth
      url: ${SGIVU_AUTH_URL:http://sgivu-auth:9000}
```

El Servicio de Clientes se comunica con el Servicio de Autenticación para la validación de tokens.

### Autenticación Interna entre Servicios

```yaml
service:
  internal:
    secret-key: "${SERVICE_INTERNAL_SECRET_KEY}"
```

<Info>
Este secreto compartido autentica las solicitudes provenientes de otros microservicios, en particular del Servicio de Compra-Venta.
</Info>

## Esquema de Base de Datos

### Archivos de Migración

```
src/main/resources/db/migration/
├── V1__create_clients_table.sql
├── V2__add_contact_information.sql
└── V3__add_client_preferences.sql
```

<Note>
Todas las migraciones se ejecutan automáticamente al iniciar. Use `validate-on-migrate: true` para garantizar la integridad del esquema.
</Note>

### Validación del Esquema

```yaml
spring:
  jpa:
    hibernate:
      ddl-auto: validate
```

<Warning>
Siempre use `validate` en producción. Nunca use `create`, `create-drop` ni `update`.
</Warning>

## Endpoints de la API

Endpoints típicos del Servicio de Clientes (accesibles a través del gateway):

```
GET    /api/clients           - List clients
GET    /api/clients/{id}      - Get client by ID
POST   /api/clients           - Create client
PUT    /api/clients/{id}      - Update client
DELETE /api/clients/{id}      - Delete client
GET    /api/clients/search    - Search clients
```

<Info>
Todos los endpoints requieren autenticación a través del flujo OAuth2 del gateway.
</Info>

## Modelo de Datos del Cliente

La entidad típica de un cliente incluye:

- Información básica (nombre, identificación, CUIT/CUIL)
- Datos de contacto (teléfono, correo electrónico, dirección)
- Preferencias y configuraciones del cliente
- Marcas de tiempo de creación/actualización
- Estado de la cuenta

## Integración con Otros Servicios

### Servicio de Compra-Venta

El Servicio de Compra-Venta consulta los datos de clientes:

```yaml
# In sgivu-purchase-sale.yml
services:
  map:
    sgivu-client:
      name: sgivu-client
      url: ${SGIVU_CLIENT_URL:http://sgivu-client:8082}
```

### Servicio de Autenticación

Valida los tokens JWT para la autorización de la API:

```yaml
services:
  map:
    sgivu-auth:
      name: sgivu-auth
      url: ${SGIVU_AUTH_URL:http://sgivu-auth:9000}
```

## Seguridad

### OAuth2 Resource Server

El servicio valida los tokens de acceso JWT emitidos por el Servicio de Autenticación.

### Protección de APIs Internas

Los endpoints internos requieren el secreto compartido:

```yaml
service:
  internal:
    secret-key: "${SERVICE_INTERNAL_SECRET_KEY}"
```

<Warning>
Mantenga la clave secreta interna segura y rótela periódicamente.
</Warning>

## Ajuste de Rendimiento

### Pool de Conexiones

Configuración predeterminada de HikariCP:

```yaml
spring:
  datasource:
    hikari:
      maximum-pool-size: 10
      minimum-idle: 5
      connection-timeout: 30000
      idle-timeout: 600000
      max-lifetime: 1800000
```

### Optimización de Consultas

- Usar `@EntityGraph` para evitar consultas N+1
- Indexar columnas consultadas frecuentemente
- Usar DTOs de proyección para operaciones de lectura
