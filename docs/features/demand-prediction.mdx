## Descripción General

La Predicción de Demanda utiliza Machine Learning para analizar los datos históricos de ventas y pronosticar la demanda futura de vehículos por segmento. Esta potente funcionalidad le permite tomar decisiones basadas en datos sobre gestión de inventario, precios y estrategias de compra.

## Cómo Funciona

El sistema de predicción de demanda de SGIVU analiza:

- Contratos de venta históricos y tendencias
- Características del vehículo (tipo, marca, modelo, año, rango de precios)
- Patrones estacionales y condiciones del mercado
- Preferencias del cliente y patrones de compra

El modelo de Machine Learning procesa estos datos para predecir la demanda esperada de diferentes segmentos de vehículos, ayudándole a optimizar la composición de su inventario.

## Obtener Predicciones

### Solicitar una Predicción

<Steps>
  <Step title="Navegar a Predicción de Demanda">
    Acceda a la sección de Predicción de Demanda desde su panel de control o el menú de analíticas.
  </Step>
  
  <Step title="Seleccionar Segmento de Vehículo">
    Elija las características del vehículo para el cual desea predecir la demanda:
    - Tipo de vehículo (auto o motocicleta)
    - Marca y modelo
    - Rango de año
    - Rango de precio
    - Otras características relevantes
  </Step>
  
  <Step title="Definir Período de Tiempo">
    Especifique el período de tiempo para su predicción (próximo mes, próximo trimestre, próximo año).
  </Step>
  
  <Step title="Generar Predicción">
    Haga clic para generar la predicción. El sistema envía sus parámetros al servicio de ML y devuelve la demanda pronosticada.
  </Step>
  
  <Step title="Revisar Resultados">
    Analice los resultados de la predicción, incluyendo:
    - Volumen de demanda esperado
    - Nivel de confianza
    - Tendencias históricas
    - Recomendaciones
  </Step>
</Steps>

### Interpretación de los Resultados

<Card title="Estimación de Demanda" icon="chart-line">
  El sistema proporciona una estimación del número de unidades que se espera vender en el segmento y período especificados. Esto se basa en patrones históricos y tendencias actuales.
</Card>

<Card title="Nivel de Confianza" icon="gauge-high">
  Cada predicción incluye una puntuación de confianza que indica qué tan fiable es el pronóstico en función de la calidad de los datos y la consistencia histórica.
</Card>

<Card title="Contexto Histórico" icon="clock-rotate-left">
  Visualice los datos de ventas históricos para el mismo segmento para comprender las tendencias y validar las predicciones contra el rendimiento pasado.
</Card>

## Entrenamiento del Modelo

El sistema de predicción mejora continuamente mediante el reentrenamiento del modelo:

### Entrenamiento Automático

<Card title="Aprendizaje Continuo" icon="brain">
  El modelo de ML se reentrena periódicamente con sus datos de ventas más recientes, asegurando que las predicciones se mantengan precisas a medida que su negocio evoluciona.
</Card>

### Reentrenamiento Manual

<Steps>
  <Step title="Solicitar Reentrenamiento">
    Acceda a la sección de gestión de modelos y solicite un reentrenamiento manual si tiene datos nuevos significativos o cambios en el mercado.
  </Step>
  
  <Step title="Especificar Rango de Datos">
    Opcionalmente, especifique un rango de fechas para los datos de entrenamiento. Por defecto, el sistema utiliza todos los datos históricos disponibles.
  </Step>
  
  <Step title="Proceso de Entrenamiento">
    El sistema reentrena el modelo utilizando los datos de contratos más recientes. Este proceso puede tardar varios minutos dependiendo del volumen de datos.
  </Step>
  
  <Step title="Validación del Modelo">
    El nuevo modelo se valida contra datos de prueba para garantizar una precisión mejorada antes de ser desplegado.
  </Step>
  
  <Step title="Despliegue">
    Una vez validado, el nuevo modelo se activa automáticamente y se utiliza para todas las predicciones futuras.
  </Step>
</Steps>

## Solución de Problemas

<AccordionGroup>
  <Accordion title="No hay predicciones disponibles">
    Esto generalmente significa que aún no se ha entrenado ningún modelo. Asegúrese de tener suficientes datos históricos de ventas y ejecute un entrenamiento del modelo.
  </Accordion>
  
  <Accordion title="Predicciones con baja confianza">
    La baja confianza puede indicar datos históricos insuficientes para el segmento específico. Intente con definiciones de segmento más amplias o acumule más historial de ventas.
  </Accordion>
  
  <Accordion title="Las predicciones parecen imprecisas">
    Compare las predicciones con los resultados reales. Si son consistentemente imprecisas, reentrene el modelo con datos más recientes o revise la calidad de sus datos de contratos.
  </Accordion>
  
  <Accordion title="El entrenamiento del modelo falla">
    Verifique que tenga datos históricos suficientes y válidos. Revise los logs del sistema o contacte a su administrador si los errores de entrenamiento persisten.
  </Accordion>
  
  <Accordion title="La solicitud de predicción expira">
    Las predicciones muy complejas pueden tardar en procesarse. Simplifique los criterios de segmento o intente nuevamente en horas de menor actividad.
  </Accordion>
</AccordionGroup>
