# Funcionalidades del Frontend

El Frontend de SGIVU está organizado en módulos de funcionalidades, cada uno encargado de un dominio específico de la aplicación. Este documento proporciona información detallada sobre cada módulo.

## Autenticación (auth)

### Descripción general

Gestiona la autenticación de usuarios, el manejo de sesiones y la autorización en toda la aplicación.

**Ubicación**: `src/app/features/auth/`

### Components

#### LoginComponent

**Path**: `/login`

Inicia el flujo de login OAuth2 redirigiendo al Endpoint de autorización del Gateway.

```typescript
startLogin(): void {
  const authUrl = `${environment.apiUrl}/oauth2/authorization/sgivu-gateway`;
  window.location.href = authUrl;
}
```

#### CallbackComponent

**Path**: `/callback`

Maneja el callback OAuth2 tras una autenticación exitosa. Verifica la sesión y redirige al Dashboard.

### Services

#### AuthService

**Responsabilidades clave:**
- Inicializar la autenticación al arrancar la aplicación
- Verificar el estado de la sesión vía `/auth/session`
- Gestionar el estado de autenticación con signals
- Manejar los flujos de login y logout
- Almacenar la información del usuario actual

**API pública:**

```typescript
class AuthService {
  // Signals (readonly)
  readonly isAuthenticated: Signal<boolean>;
  readonly isDoneLoading: Signal<boolean>;
  readonly currentAuthenticatedUser: Signal<User | null>;
  readonly isAdmin: Signal<boolean>;
  
  // Methods
  initializeAuthentication(): Promise<void>;
  login(): void;
  logout(): void;
  checkSession(): Observable<AuthSessionResponse>;
}
```

#### PermissionService

**Responsabilidades clave:**
- Verificar permisos de usuario
- Evaluar acceso basado en roles
- Soportar Guards de permisos

**API pública:**

```typescript
class PermissionService {
  hasPermission(permission: string): boolean;
  hasAnyPermission(permissions: string[]): boolean;
  hasAllPermissions(permissions: string[]): boolean;
  hasRole(role: string): boolean;
}
```

### Guards

#### authGuard

Protege rutas que requieren autenticación:

```typescript
{
  path: 'dashboard',
  canActivate: [authGuard],
  loadComponent: () => import('./dashboard.component')
}
```

#### permissionGuard

Protege rutas que requieren permisos específicos:

```typescript
{
  path: 'roles-permissions',
  canActivate: [authGuard, permissionGuard],
  data: {
    canActivateFn: (ps: PermissionService) => ps.hasPermission('role:update')
  }
}
```

### Interceptors

#### defaultOAuthInterceptor

- Agrega `withCredentials: true` a las peticiones API
- Habilita la gestión de sesiones basada en cookies
- Maneja errores 401 por sesiones expiradas

### Directives

#### hasPermissionDirective

Muestra/oculta elementos condicionalmente según los permisos:

```html
<button *hasPermission="'vehicle:delete'" (click)="deleteVehicle()">
  Delete
</button>
```

## Dashboard

### Descripción general

Proporciona analíticas, KPIs y visualizaciones de métricas de negocio.

**Ubicación**: `src/app/features/dashboard/`

### Components

#### DashboardComponent

**Path**: `/dashboard`

Dashboard principal con:
- Tarjetas KPI (usuarios totales, vehículos, contratos activos, ingresos)
- Gráfico de distribución de estados de vehículos
- Gráfico de tendencia de ventas
- Información de predicción de demanda
- Feed de actividad reciente

**Características:**
- Actualizaciones de datos en tiempo real
- Visualizaciones con Chart.js
- Diseño de grilla responsivo
- Visibilidad de contenido basada en permisos

### Services

#### DashboardService

Obtiene métricas del Dashboard:

```typescript
class DashboardService {
  getKPIs(): Observable<DashboardKPIs>;
  getVehicleStatusDistribution(): Observable<StatusDistribution[]>;
  getSalesTrend(startDate: Date, endDate: Date): Observable<SalesTrend[]>;
  getRecentActivity(): Observable<Activity[]>;
}
```

### Utilidades

- Helpers de configuración de gráficos
- Utilidades de rango de fechas
- Formateadores de KPIs

## Gestión de Usuarios (users)

### Descripción general

Gestiona los usuarios de la aplicación, roles y permisos.

**Ubicación**: `src/app/features/users/`

**Routes**: `/users/*`

### Components

#### UserListComponent

**Path**: `/users`

Muestra una lista paginada de usuarios con:
- Búsqueda y filtrado
- Ordenamiento por columna
- Insignias de rol
- Acciones rápidas (editar, eliminar, ver perfil)
- Visibilidad de acciones basada en permisos

#### UserFormComponent

**Paths**: `/users/create`, `/users/:id/edit`

Formulario para crear/editar usuarios:
- Información personal (nombre, email, teléfono)
- Asignación de roles
- Información de dirección
- Validación de formulario
- Manejo de éxito/error

#### UserProfileComponent

**Path**: `/users/:id`

Vista detallada del perfil de usuario:
- Información personal
- Roles y permisos asignados
- Historial de actividad
- Botón de editar perfil

#### RolesPermissionsComponent

**Path**: `/roles-permissions`

Gestión de roles y permisos del sistema:
- Listar todos los roles
- Asignar permisos a roles
- Crear/editar roles
- Ver matriz de permisos

**Permiso requerido**: `role:update`

### Services

#### UserService

Operaciones CRUD para usuarios:

```typescript
class UserService {
  getUsers(params?: HttpParams): Observable<PaginatedResponse<User>>;
  getUser(id: string): Observable<User>;
  createUser(user: User): Observable<User>;
  updateUser(id: string, user: User): Observable<User>;
  deleteUser(id: string): Observable<void>;
  getUserCount(): Observable<UserCount>;
}
```

### Models

```typescript
interface User {
  id: string;
  username: string;
  email: string;
  person: Person;
  roles: Role[];
  enabled: boolean;
  createdAt: Date;
  updatedAt: Date;
}

interface Person {
  id: string;
  firstName: string;
  lastName: string;
  documentType: string;
  documentNumber: string;
  phone?: string;
  address?: Address;
}

interface Role {
  id: string;
  name: string;
  permissions: Permission[];
}
```

### Resolvers

#### userProfileResolver

Precarga los datos del usuario antes de activar la ruta:

```typescript
{
  path: ':id',
  component: UserProfileComponent,
  resolve: { user: userProfileResolver }
}
```

## Inventario de Vehículos (vehicles)

### Descripción general

Gestiona el inventario de vehículos incluyendo autos y motocicletas.

**Ubicación**: `src/app/features/vehicles/`

**Routes**: `/vehicles/*`

### Components

#### VehicleListComponent

**Path**: `/vehicles`

Lista combinada de todos los vehículos:
- Vista con pestañas (Todos, Autos, Motocicletas)
- Filtros de estado (Disponible, Reservado, Vendido)
- Búsqueda por marca/modelo/VIN
- Miniaturas de imágenes
- Visualización de precio
- Acciones rápidas

#### CarListComponent

**Path**: `/vehicles/cars`

Listado específico de autos:
- Atributos específicos de autos (puertas, asientos)
- Filtrado por rango de precio
- Filtrado por año
- Tipo de transmisión

#### MotorcycleListComponent

**Path**: `/vehicles/motorcycles`

Listado específico de motocicletas:
- Cilindrada del motor
- Tipo de motocicleta (sport, cruiser, etc.)
- Filtrado por rango de precio

#### VehicleFormComponent

**Paths**: `/vehicles/create`, `/vehicles/:id/edit`

Formulario para crear/editar vehículos:
- Selección de tipo de vehículo
- Información básica (marca, modelo, año, VIN)
- Especificaciones (motor, transmisión, kilometraje)
- Precio
- Gestión de estado
- Carga de imágenes
- Validación

#### VehicleDetailComponent

**Path**: `/vehicles/:id`

Vista detallada del vehículo:
- Especificaciones completas
- Galería de imágenes

### Flujo de carga de imágenes

1. **Solicitar URL prefirmada**:
   ```typescript
   vehicleImageService.requestPresignedUploadUrl({
     vehicleId: '123',
     fileName: 'car.jpg',
     contentType: 'image/jpeg'
   });
   ```

2. **Cargar a S3**:
   ```typescript
   vehicleImageUploadService.uploadToS3(presignedUrl, file);
   ```

3. **Confirmar carga**:
   ```typescript
   vehicleImageService.confirmUpload(vehicleId, {
     s3Key: 's3-object-key',
     fileName: 'car.jpg'
   });
   ```

## Matriz de permisos

### Permisos de usuario

- `user:read` - Ver usuarios
- `user:create` - Crear usuarios
- `user:update` - Editar usuarios
- `user:delete` - Eliminar usuarios

### Permisos de cliente

- `client:read` - Ver clientes
- `client:create` - Crear clientes
- `client:update` - Editar clientes
- `client:delete` - Eliminar clientes

### Permisos de vehículo

- `vehicle:read` - Ver vehículos
- `vehicle:create` - Crear vehículos
- `vehicle:update` - Editar vehículos
- `vehicle:delete` - Eliminar vehículos
- `vehicle:manage_images` - Cargar/eliminar imágenes

### Permisos de transacción

- `transaction:read` - Ver transacciones
- `transaction:create` - Crear transacciones
- `transaction:update` - Editar transacciones
- `transaction:cancel` - Cancelar transacciones
- `transaction:report` - Generar reportes

### Permisos de rol

- `role:read` - Ver roles
- `role:update` - Gestionar roles y permisos
